今天，“Mounts of Mayhem（群骑纷争）”强势登陆 Minecraft，带来全新的可操使武器、可驯服坐骑、强敌对手与作战策略！在炽烈的阳光下、夜空之中、甚至深海之底，都有挑战者等待与你交锋——每一次对抗都需要全新的策略。拿起“矛”这把全新的分阶武器，用你的准度与坐骑的速度来造成伤害；上界的骑手们——整装待发，率领坐骑冲锋，迎接群骑纷争更新！

## 新特性

- 新增鹦鹉螺与僵尸鹦鹉螺
- 新增鹦鹉螺甲
- 新增矛武器
- 更新了一些进度以包含鹦鹉螺、僵尸鹦鹉螺、僵尸马、骆驼尸壳与焦骸
- 新增图形选项“纹理过滤”与“各向异性过滤”
- 新增下界合金马铠，可在锻造台将钻石马铠升级获得

### 鹦鹉螺

- 鹦鹉螺是一种新的中立水生生物
- 在所有海洋群系中生成
- 受激时会使用冲刺攻击
- 偶尔会对范围内的河豚发起冲刺攻击
- 具有类似骆驼的冲刺技能，按跳跃键触发
- 上岸会因窒息受到伤害
- 会为骑乘的玩家添加“鹦鹉螺之息”效果
- 可使用河豚或一桶河豚进行繁殖与驯服
- 会自然消失，但与玩家互动后会变为持久
- 被驯服的鹦鹉螺：
  - 可用鞍进行骑乘
  - 装有鞍时游荡半径为 16 格
  - 未装鞍时游荡半径为 32 格

#### 僵尸鹦鹉螺

- 僵尸鹦鹉螺是一种新的水下生物，会与持三叉戟的溺尸骑手一起生成
- 只有在被敌对生物骑乘时才会敌对
- 行为与鹦鹉螺类似，但无法繁殖

珊瑚僵尸鹦鹉螺 —— 僵尸鹦鹉螺拥有一种珊瑚变种，会在暖水海洋群系生成

### 鹦鹉螺甲

- 可供鹦鹉螺与僵尸鹦鹉螺装备
- 提供铜、金、铁、钻石与下界合金版本

### 焦骸

- 新的骷髅变种
- 在沙漠中于光照等级 0 生成，替代部分普通骷髅
- 不会在阳光下燃烧
- 射出虚弱之箭
- 免疫虚弱效果
- 射速相比普通骷髅更慢，类似沼骸
- 不在洞穴生成，遵循尸壳的生成规则

### 骆驼尸壳

- 新的亡灵骆驼变种
- 在沙漠中于光照等级 0 处生成
- 不会在阳光下燃烧
- 生成时乘有两名骑手：持矛的尸壳与焦骸
- 不在洞穴中生成，遵循尸壳的生成规则
- 若骑手为敌对生物则敌对
- 无骑手时为被动
- 随其他敌对生物一同自然消失，但与玩家互动后会变为持久
- 最爱食物是兔子脚
- 无法繁殖
- 作为坐骑的行为与普通骆驼相同
- 被杀死时掉落 2–3 个腐肉，每级抢夺额外掉落 0–1 个

### 矛

- 新武器，可制作为木、石、铜、铁、金、钻石与下界合金版本
- 不同材料拥有不同的计时属性，影响可用速度
- 具有最小伤害距离，过近将不会造成伤害
- 相比其他工具与武器具有更长的最大攻击距离
- 命中时会有回弹的视觉效果
- 拥有两种攻击：戳刺与冲锋
- 不会触发物品交互振动
- 僵尸、流浪者、僵尸化猪灵与猪灵可携带矛生成
- 金矛被猪灵视为偏好武器

#### 戳刺

- 快速按下主操作键以使用
- 低伤害并带击退
- 攻击之间有冷却
- 木矛冷却最快；下界合金最慢
- 可命中多个敌人
- 不会破坏方块

#### 冲锋攻击

- 按住次要操作键使用
- 伤害基于矛材质、玩家视角以及玩家与目标的速度
- 按住期间，攻击会经历三个阶段后回到空闲：
  - 发动：若速度超过阈值，矛可造成伤害、击退并使被骑乘的实体下马
  - 疲惫：矛竖直并抖动以示，此阶段若速度超过阈值可造成伤害与击退（但不下马）
  - 脱离：矛下垂指向下方以示，此阶段若速度超过阈值只造成伤害，不产生击退或下马
- 生物使用矛时的攻击距离为玩家的一半，但其造成伤害、下马与击退所需速度阈值更低
- 僵尸、流浪者、僵尸化猪灵与猪灵会使用矛进行冲锋

#### 突进魔咒

- 矛专属附魔
- 仅对戳刺攻击生效
- 使用突进进行戳刺时，玩家会沿视线方向水平推进
- 视角需完全水平以获得最大突进距离
- 使用突进会消耗 1 点耐久，且与附魔等级无关
- 使用突进会通过增加玩家疲劳来消耗饥饿值，类似奔跑或跳跃
  - 1 级消耗 1 点饥饿；2 级消耗 2 点；3 级消耗 3 点
  - 饥饿槽至少有 6 点时才能使用突进
- 在水中或用鞘翅飞行时不生效

### 纹理过滤

- 视频设置中的新选项
- 控制以斜角或远距离观看纹理时的采样方式
- 可选值为 “无”、“RGSS” 与 “各向异性”
- 默认值取决于图形预设与硬件
- “RGSS”是一种基于着色器的过滤方式，在中等性能开销下提升纹理清晰度
  - 在所有硬件上可用
- “各向异性” 是一种基于硬件的过滤方式，提供最佳纹理清晰度
  - 最大各向异性可为 “2x”、“4x”、“8x”，由单独的滑块控制
  - 如硬件不支持，各向异性过滤不会产生效果
  - 这会显著增加显存占用（尤其在 mipmap 层数较高时）

  * 速率限制为每 30 秒 1 次通知
* 在网页浏览器中启用验证
  * 当打开 WebSocket 连接时通过在 `Sec-WebSocket-Protocol` 头中传递 token 验证
    * 例如：`Sec-WebSocket-Protocol: minecraft-v1, <token>`
  * 以这种方式进行验证的请求会受到 Origin 头检查。为此需要在服务器端通过配置 `management-server-allowed-origins` 指定允许的来源。默认值为空，这意味着基于 `Sec-Websocket-Protocol` 的验证实际上被禁用
* 在 `typed_game_rule` 与 `untyped_game_rule` 模式中，`value` 字段的类型由字符串更改为可取布尔值或整型值

### 僵尸鹦鹉螺变种

* 通过在 `data/<namespace>/zombie_nautilus_variant/<id>.json` 中添加条目，可以以数据驱动方式定义僵尸鹦鹉螺变种
* 本特性为实验性
* 文件字段：
  * `model` —— 取值之一：`normal`、`warm`
  * `asset_id` —— 此变种资源的命名空间 ID，解析为 `assets/<namespace>/textures/<path>.png`

#### 生成条件

### 时间线

时间线是一个新的实验性概念，定义在数据包的 `timeline` 注册表下（即位于 `data/<namespace>/timeline/<id>.json`）。

时间线基于绝对昼夜时间（即 `/time set` 所设置的时间）来指定游戏行为。时间线可选地指定一个周期，使之循环重复。

时间线的主要用途是作为环境属性的一个来源，从而对环境属性应用修饰符，以修改游戏行为或视觉效果。

格式：包含以下字段的对象

* `period_ticks`：可选整数，定义时间线的循环周期刻数
  * 若未指定，则该时间线不循环
* `tracks`：可选映射，将环境属性 ID 映射到对应的环境属性轨迹对象
  * 详见下文环境属性相关章节

时间线默认不会生效，除非被某个维度引用 —— 见下文“维度类型”。

### 环境属性

环境属性提供了一种数据驱动的方式来控制多种视觉与游戏性系统。

每个环境属性控制一个特定的视觉或游戏效果：例如，`minecraft:visual/sky_color` 控制天空的颜色，而 `minecraft:gameplay/water_evaporates` 控制某位置是否可以放置水。

> [!DEVNOTE] 该系统及本次引入的许多属性应视为高度实验性，它们在后续快照与正式版本中仍可能发生显著变化。与此同时，一如既往，我们期待并欢迎你提出任何反馈或建议！


#### 来源

环境属性的数值可由以下环境属性来源提供（按优先级从低到高列出）：

* 维度
* 生物群系
* 时间线
* 天气（未数据驱动）

生物群系作为来源会基于位置修改属性，时间线则可基于时间修改属性。

环境属性的“有效”数值（即游戏中实际呈现的）将根据各来源的优先级，对它们提供的数值进行组合。

例如，以下情形：

* 主世界维度提供 `sky_color = #00ff00`（绿色）
* 平原生物群系提供 `sky_color = #ff0000`（红色）

当玩家处于平原群系时，会看到红色的 `sky_color`；而在主世界的其他位置，则看到绿色。

当某个环境属性来源提供一个环境属性时，它可以：

* 覆盖该数值，如上例中平原生物群系覆盖了主世界的 `sky_color`
* 对之前的数值应用一个修饰符（见下文“修饰符”章节）

#### 修饰符

如上例所述，一个环境属性来源可以直接覆盖某属性数值。然而，有时也希望对低优先级来源提供的数值施加一个修饰符，而非直接覆盖。

例如，以下情形：

* 主世界维度提供 `water_fog_end_distance = 96.0`
* 平原生物群系使用乘法修饰符（multiply 0.85）修改 `water_fog_end_distance`

当处于平原时，`water_fog_end_distance` 解析为 `96.0 * 0.85 = 81.6`；在主世界的任何其它群系则解析为 `96.0`。

可用的修饰符类型取决于环境属性的类型。最基础且每个环境属性都支持的修饰符是 `override`，它仅作为对前一个数值的覆盖。若来源未显式指定修饰符，则默认为 `override`。

所有可用修饰符及其行为的完整描述见下文“通用修饰符”章节。

#### 插值

修饰符描述的是某来源提供的数值如何作用于先前来源的数值；而“插值”描述的是同一个来源内部多个数值之间的组合。仅部分环境属性支持插值。这允许在两个或多个数值之间进行平滑过渡，例如在两个生物群系之间移动时。

在多个生物群系之间，插值作用于来源的数值，且发生在修饰符应用之后；在时间线中，插值则作用于修饰符的参数，且发生在修饰符应用之前。

例如，以下情形：

* 平原群系提供 `sky_color = #ff0000`（红）
* 沙漠群系提供 `sky_color = #ffff00`（黄）

`sky_color` 就是一个会基于位置平滑过渡的属性：在上述情形中，玩家从平原移动到沙漠时，天空颜色会逐渐从红转为黄。而其它属性，如 `water_evaporates`，表示离散值，不会进行平滑过渡 —— 只考虑玩家所在精确位置的那个群系。

生物群系之间的平滑过渡基于相机周围 8 格半径内的生物群系。占据该半径更大体积且更接近相机的群系，对最终插值结果影响更大。

时间线可以定义属性轨道——通过关键帧来指定某个环境属性的修饰器随时间如何变化。根据具体属性，关键帧之间可使用插值；若不使用插值，则始终选取上一个关键帧的取值。

例如，周期为 `24000` 的时间线可如下设置：

- 在时间 `0`：提供 `sky_color = #ff0000`（红）
- 在时间 `1000`：提供 `sky_color = #ff0000`（红）
- 在时间 `6000`：提供 `sky_color = #ff00ff`（品红）

在时间 `0` 到 `1000` 之间，天空颜色为红色；在 `1000` 到 `6000` 之间，颜色将从红逐渐过渡到品红；随后，从 `6000` 开始直到时间线重复回到 `0`，颜色会缓慢回到红色。

注意：与生物群系不同，若时间线使用的是修饰器而非覆盖，则插值应用在修饰器的参数上，而非最终被修饰后的取值。

#### 环境属性映射

维度类型与生物群系定义包含一个新的 `attributes` 字段，允许它们定义环境属性。

该映射通常是一个对象，将环境属性 ID 映射到其对应取值，例如：

```JSON
"attributes": {
    "minecraft:visual/fog_color": "#ffaa00",
    "minecraft:gameplay/water_evaporates": true
}
```

像上面这样直接定义的值，总是假定使用 `override` 修饰符。

不过，也可以展开值对象以表达不同的修饰符，格式为一个对象，字段如下：

* `modifier` —— 可选 字符串修饰符 ID，依属性类型而定（见下文“通用修饰符”）
  * 默认：`override`
* `argument` —— 修饰符参数（格式取决于所选修饰符）
  * 参数的使用方式也取决于修饰符类型

例如，以下定义表示将 `water_fog_end_distance` 乘以 85%：

```JSON
"attributes": {
    "minecraft:visual/water_fog_end_distance": {
        "modifier": "multiply",
        "argument": 0.85
    }
}
```

#### 属性轨迹

时间线中的属性轨迹为如下字段的对象：

* `ease` —— 可选缓动类型，用于在关键帧之间对数值插值进行缓动（见下文）
  * 默认：`linear`
  * 若目标属性不支持插值，则缓动类型不生效
* `keyframes` —— 关键帧对象列表，必须按 `ticks` 升序：
  * `ticks` —— 整数，位于 `0` 到 `period_ticks`（若已指定）之间，表示该关键帧生效的刻
  * `value` —— 修饰符参数（格式取决于所选修饰符）
    * 若未指定 `modifier`（或为 `override`），该字段类型与环境属性本身一致
    * 此值如何使用取决于修饰符类型
  * 注意：同一刻最多可放置两个关键帧，以实现瞬时切换
* `modifier` —— 可选 字符串修饰符 ID，依属性类型而定（见下文“通用修饰符”）
  * 默认：`override`

例如，下面这个时间线的周期为 `24000`，仅包含一个对 `minecraft:gameplay/cat_waking_up_gift_chance` 进行修改的属性轨迹。由于缓动类型为 `constant`，因此在 `362` 到 `23667` 之间取值为 `0.0`，而在 `23667` 到下一个 `362` 之间取值为 `0.7`。

与其他环境属性来源一样，轨迹中指定的数值会作为 `modifier` 的参数，叠加在较低优先级来源之上。本例中修饰符为 `maximum`，因此该时间线只会把 `cat_waking_up_gift_chance` 的取值提高到轨迹所给的数值（不会降低）。

```JSON
{
    "period_ticks": 24000,
    "tracks": {
        "minecraft:gameplay/cat_waking_up_gift_chance": {
            "ease": "constant",
            "modifier": "maximum",
            "keyframes": [
                { "ticks": 362,   "value": 0.0 },
                { "ticks": 23667, "value": 0.7 }
            ]
        }
    }
}
```

#### 通用修饰符

每个环境属性都有一个特定的“值类型”，它描述数值必须如何定义、支持哪些修饰符，以及如何插值。有些类型非常专用，另一些则被多个属性复用。

虽然下表并不穷尽，但以下属性类型在很多环境属性中常用：

##### 布尔修饰符

参数格式：布尔值

- `override`
- `and`
- `nand`
- `or`
- `nor`
- `xor`
- `xnor`

##### 浮点修饰符

参数格式：float

- `override`
- `add`
- `subtract`
- `multiply`
- `minimum`
- `maximum`
- `alpha_blend`
  * 按公式 `result = lerp(alpha, subject, value)` 修改 `subject`
  * 参数格式：对象，字段如下：
    * `value`：要混合趋近的浮点数
    * `alpha`：位于 `0` 与 `1` 之间的浮点数

##### 颜色修饰符
参数格式：RGB 颜色（`alpha_blend` 例外）

- `override`
  * 参数格式：RGB 颜色
- `add` —— 分量逐项相加的颜色混合
  * 参数格式：RGB 颜色
- `subtract` —— 分量逐项相减的颜色混合
  * 参数格式：RGB 颜色
- `multiply` —— 分量逐项相乘的颜色混合
  * 参数格式：RGB 颜色
  * 若提供 ARGB，则 α 分量会与其他颜色通道一样独立相乘
- `alpha_blend` —— 常规 α 混合（图像编辑软件常见）
  * 参数格式：ARGB 颜色
  * 当参数 α 为 `1` 时，行为等同 override（不进行混合）
- `blend_to_gray` —— 先将颜色转为灰度，应用亮度修饰，再按因子进行混合
  * 其中 `gray = brightness * (0.3 * red + 0.59 * green + 0.11 * blue)`，`result = lerp(factor, subject, [gray, gray, gray])`
  * 参数格式：对象，字段如下：
    * `brightness`：位于 `0` 与 `1` 之间的浮点数，作用于灰度的乘数
    * `factor`：位于 `0` 与 `1` 之间的浮点数，用于混合的因子

##### ARGB 颜色修饰符

- `override`
  * 参数格式：ARGB 颜色
- `add` —— 分量逐项相加的颜色混合
  * 参数格式：RGB 颜色
- `subtract` —— 分量逐项相减的颜色混合
  * 参数格式：RGB 颜色
- `multiply` —— 分量逐项相乘的颜色混合
  * 参数格式：RGB 或 ARGB 颜色
  * 若指定，则 α 分量会与其他颜色通道一样独立相乘
- `alpha_blend` —— 常规 α 混合（图像编辑软件常见）
  * 参数格式：ARGB 颜色
  * 当参数 α 为 `1` 时，行为等同 override（不进行混合）
- `blend_to_gray` —— 与 RGB 颜色的行为相同

#### 新增 `minecraft:attack_range`

- 使用该物品时启用自定义攻击范围。
- 覆盖玩家的常规实体交互范围。
- 常规近战攻击范围基于目标碰撞箱与攻击者碰撞箱的距离计算。此组件通过调整攻击碰撞箱的最小和最大边距来修改该距离校验。
- 使用 `minecraft:kinetic_weapon` 的生物，其攻击范围的应用方式与玩家相同。
- 格式：对象，包含以下字段：
  - `min_reach`：浮点数，从攻击者到目标的最小有效距离（方块）
    - 默认值：`0.0`，有效范围 `0.0` 到 `64.0`
  - `max_reach`：浮点数，从攻击者到目标的最大有效距离（方块）
    - 默认值：`3.0`，有效范围 `0.0` 到 `64.0`
  - `min_creative_reach`：浮点数，创造模式攻击者到目标的最小有效距离（方块）
    - 默认值：`0.0`，有效范围 `0.0` 到 `64.0`
  - `max_creative_reach`：浮点数，创造模式攻击者到目标的最大有效距离（方块）
    - 默认值：`5.0`，有效范围 `0.0` 到 `64.0`
  - `hitbox_margin`：浮点数，校验有效碰撞时应用于目标边界框的边距
    - 默认值：`0.3`，有效范围 `0.0` 到 `1.0`
  - `mob_factor`：浮点数，当物品由生物使用时，应用于 `min_range` 与 `max_range` 的距离倍率
    - 默认值：`1.0`，有效范围 `0.0` 到 `2.0`

#### 新增 `minecraft:kinetic_weapon`

- 使用该物品时启用蓄力型攻击（主要用于矛类）。在使用过程中，每当有实体接触到攻击射线时都会造成伤害，伤害与实体的相对速度相关。
- 格式：对象，包含以下字段：
  - `contact_cooldown_ticks`：整数，命中并与实体失去接触后，重新能够命中的冷却刻数
    - 默认值：`10`
  - `delay_ticks`：整数，武器生效前所需的延迟刻数
    - 默认值：`0`
  - `dismount_conditions`、`knockback_conditions`、`damage_conditions`：分别用于应用落马、击退与伤害效果的条件。对象字段：
    - `max_duration_ticks`：整数，超过该刻数后不再检查条件。计时从 `delay` 结束后开始
    - `min_speed`：浮点数，攻击者沿其视线方向的最小速度（方块/秒）
      - 可选，默认值：`0.0`
    - `min_relative_speed`：浮点数，攻击者与目标沿攻击者视线方向的最小相对速度（方块/秒）
      - 可选，默认值：`0.0`
  - `forward_movement`：浮点数，动画期间物品从手中向前移动的距离
    - 默认值：`0.0`
  - `damage_multiplier`：浮点数，基于相对速度的最终伤害倍率
    - 默认值：`1.0`
  - `sound`：可选，武器进入蓄力时播放的音效事件
  - `hit_sound`：可选，武器命中实体时播放的音效事件
  - 示例：`kinetic_weapon={forward_movement:1.0,delay:20,damage_conditions:{max_duration:60},knockback_conditions:{max_duration:40},dismount_conditions:{max_duration:20}}`
- 伤害计算方式为 `floor(relative_speed * velocity_multiplier)`，其中 `relative_speed` 为攻击者与目标的速度向量在攻击者视线轴上的投影差值。
  - 附魔或属性修饰符带来的额外伤害在此计算之后再叠加

### 槽位源

- 新增槽位源，使数据包中可以指定任意物品栏槽位的位置。
- 格式：对象，包含以下字段：
  - `type`：槽位源类型
  - `<type-specific>`：根据类型而定的附加字段

#### `minecraft:empty` 类型

- 空选择，不包含任何槽位

#### `minecraft:group` 类型

- 将多个槽位源合并为一个，结果选择包含各来源提供的全部槽位。
  - 若某槽位出现在多个来源中，它将在结果中重复出现
  - 例如 `[a, b] + [c, a]` -> `[a, b, c, a]`
- 格式：
  - `terms`：要合并的槽位源列表
- 也可直接以内联的槽位源列表书写

#### `minecraft:slot_range` 类型

- 从实体或方块实体的物品栏中选择一个槽位范围内的槽位。
- 行为与 `/item` 命令的 `from` 参数一致。
- 格式：
  - `source`：从战利品上下文中获取槽位的实体或方块实体
    - 可为 `block_entity`、`this`、`attacking_entity`、`last_damage_player`、`direct_attacker`、`target_entity` 或 `interacting_entity`
  - `slots`：槽位范围，格式为 `<slot_type>` 或 `<slot_type>.<slot_number>`（例如 `armor.chest` 或 `container.*`）

#### `minecraft:contents` 类型

- 从一个或多个物品的物品栏组件中选择所有非空槽位。
  - 若该组件中未存放物品，结果选择为空
- 待使用其物品栏组件的物品位置由另一个槽位源指定。
  - 若该槽位源包含多个带该组件的物品，结果选择将与 `minecraft:group` 类型相同地合并
  - 例如：`束口袋 [a, b] + 潜影盒 [c, d]` -> `[a, b, c, d]`
- 格式：
  - `component`：要定位的物品栏组件
    - 允许值为 `minecraft:bundle_contents`、`minecraft:charged_projectiles` 与 `minecraft:container`
  - `slot_source`：包含待定位物品槽位的槽位源

#### `minecraft:filtered` 类型

- 对已选槽位应用过滤器，将不匹配的槽位从结果中排除。
- 格式：
  - `item_filter`：对各槽位中的物品进行匹配的物品谓词
  - `slot_source`：需要过滤的槽位源

#### `minecraft:limit_slots` 类型

- 限制提供的槽位数量，结果选择至多包含该数量的槽位。
  - 超出限制的槽位将按包含顺序被排除
  - 例如 `[a, b, c, d]` 在限制为 `3` 时 -> `[a, b, c]`
- 格式：
  - `limit`：整数，结果选择中允许包含的最大槽位数
  - `slot_source`：需要施加限制的槽位源

示例：从实体的快捷栏与护甲槽中选择所有物品数量大于 16 的槽位：

```JSON
{
    "type": "minecraft:filtered",
    "item_filter": {
        "count": {
            "min": 16
        }
    },
    "slot_source": [
        {
            "type": "minecraft:slot_range",
            "source": "this",
            "slots": "hotbar.*"
        },
        {
            "type": "minecraft:slot_range",
            "source": "this",
            "slots": "armor.*"
        }
    ]
}
```

### 谓词

#### 组件谓词

- 组件谓词（方块、物品与实体谓词中的 `predicates` 字段）现在包含用于检查各组件类型存在性的谓词。
- 这类谓词写作 `{<component_type>: {}}`。
- 只要组件存在，该检查即视为通过，实际值不影响结果。
- 针对特定组件的既有谓词保持不变。
- 示例：
  - `{predicates:{written_book_content:{author:"foo"}}` —— 既有的书籍匹配格式
  - `{predicates:{written_book_content:{}}` —— 上述的特例，只要组件存在即接受任意值
  - `{predicates:{instrument:{}}` —— 检查组件 `minecraft:instrument` 是否存在（此前不允许）

- 与数据中的组件谓词相同，这些空条目仅检查组件是否存在。
- 用于检查组件存在性的既有简写语法保持不变。
  - 即 `*[instrument]` 与 `*[instrument~{}]` 等效

## 纹理

- `*.mcmeta` 文件的 `texture` 部分新增两个字段：`mipmap_strategy` 与 `alpha_cutoff_bias`
  - `mipmap_strategy` 支持以下取值：
    - `mean`：在当前 mipmap 层级上对每 4 像素分组的颜色与透明度求平均以生成下一层像素。多数实心或半透明整方块纹理的默认策略，沿用不变
    - `dark_cutout`：与 `mean` 类似，但以一种使裁剪像素边界更暗的方式混合颜色。用于树叶与红树根，沿用不变。其用于模拟方块内部更暗的效果
    - `cutout`：新策略，总是基于原始纹理而非上一层 mipmap 来生成下一层。用于本版本中大多数新增支持 mipmap 的裁剪类方块
    - `strict_cutout`：`cutout` 的变体，使用更严格的透明度裁剪值，使得使用该值的纹理在更高 mipmap 层级上会消失。用于花朵等以避免伪影
    - `auto`：默认值；对于不含完全透明像素的纹理选择 `mean`，含完全透明像素的选择 `cutout`
  - `alpha_cutoff_bias`：浮点字段，用于控制裁剪纹理的透明度偏置
    - 技术上没有限制，可为负数；但超过 `1.0` 或低于 `-1.0` 意义不大，因为纹理将分别变为完全不透明或完全透明
    - 默认值为 `0.0`；可在远处纹理过于透明/纤薄时适当增大，或在过于“不透明”时减小
    - 建议小幅调整以寻找最佳点。例如海带（`kelp`）纹理使用 `0.1`，防止远处完全透明
    - 仅用于裁剪纹理的低层 mip，不改变首层 mip 的透明度，对其他纹理无效
- 玻璃、玻璃板与红石粉现支持半透明纹理

### 物品模型

- 为所有物品模型新增选项：`swap_animation_scale`
  - 默认值：`1.0`
  - 表示在快捷栏切换物品时，物品上下移动的速度
  - 较大的速度可使占屏空间更大的物品在切换到下一个物品前完全下沉

- `spear_in_hand` —— 手持矛时的模型

### 方块模型

- 方块模型元素现可绕多个轴旋转。
  - 模型先绕 X，再绕 Y，最后绕 Z 旋转
  - 新增字段：
    - `x`：绕 X 轴旋转角度（度），浮点数，默认 `0.0`
    - `y`：绕 Y 轴旋转角度（度），浮点数，默认 `0.0`
    - `z`：绕 Z 轴旋转角度（度），浮点数，默认 `0.0`
  - 既有字段 `axis` 与 `angle` 仍可使用
    - 若同时存在两套字段，旧式写法优先生效
- 移除了角度值限制 `[-45, 45]`。
  - 注意：各面的名称不受旋转影响。依赖它们的特性（如裁剪）始终使用原始方向

### 方块状态模型派发

- 方块状态派发文件（定义于 `assets/.../blockstates/`）中的变体现也可绕 Z 轴旋转。
- 新增可选字段 `z`，格式与既有的 `x` 与 `y` 字段相同：整数，允许值 `0`（默认）、`90`、`180` 与 `270`
- 绕 Z 轴的旋转在 X 与 Y 之后应用

### 着色器与后处理效果

- 新增 `animate_sprite*` 核心着色器，使用新的 `SpriteAnimationInfo` uniform

#### 精灵图动画

- 精灵图（作为更大图集的一部分的纹理）现于 GPU 上进行动画，而非每刻由 CPU 处理。
- 常规帧动画使用 `animate_sprite.vsh` 与 `animate_sprite_blit.fsh` 进行绘制。
- 插值动画使用 `animate_sprite.vsh` 与 `animate_sprite_interpolate.fsh` 进行绘制。
- UBO `SpriteAnimationInfo` 包含精灵图在大纹理图集中应绘制到何处的信息。
