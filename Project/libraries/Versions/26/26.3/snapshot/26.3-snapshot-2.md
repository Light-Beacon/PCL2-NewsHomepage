## 已知问题

- 在 macOS 上使用 Vulkan 时，世界不会渲染，屏幕显示为粉红色。
- 实体的光照显示不正确，例如顶面渲染得过暗。

## 更改

- 持久实体的随机漫游/游泳行为现在将在玩家不在附近时停用，与非持久实体的处理方式一致。
- 将当前的“改良透明”实现替换为一种新的与顺序无关的透明度算法（order-independent transparency，简称 OIT），预计在解决半透明渲染问题上有更好效果。
- 当玩家同时持有盾牌与锄或铁锹时，举起盾牌将优先于耕地与铺路动作。

### 对方块、物品与实体的微小调整

- 手持三叉戟的溺尸在玩家距离 3 格以内将改为使用近战攻击。

### 与顺序无关的透明度（OIT）

- 通常，半透明表面需要按从远到近的顺序渲染以避免视觉问题：
  - 为此，半透明几何体在渲染前必须被排序；
  - 这非常耗费性能，即便对单三角形进行完美排序，交叉的三角形仍然无法正确渲染。
- 在此快照之前，“改良透明度”设置通过把不同类型的半透明对象（实体、粒子、水等）渲染到不同的纹理，然后合成最终图像，来缓解透明度相关的视觉问题：
  - 这种方法修复了一些问题，但带来了性能开销；
  - 同时也引入了一些不太明显的其他问题。
- 现在，“改良透明度”选项将改为启用一种新的与顺序无关的透明度算法（OIT）：
  - OIT 使得可以在任意顺序下渲染半透明对象而无需排序；
  - 它修复了现有的大多数半透明相关的主要视觉问题（例如通过半透明物体观察时某些对象消失）；
  - 但该算法是近似的，因此会引入一些新的轻微视觉伪影（例如通过多层半透明表面观察时，云的颜色可能有些不正确）。
- 我们预计 OIT 在性能上的影响会高于之前的“改良透明度”实现。
- 当关闭“改良透明度”时，其行为并未发生实质性变化，游戏在必要时仍会优先考虑性能而对几何体进行排序。

> [!DEVNOTE] 特别感谢 Rubén Osorio López、Jozufozu 以及常规模组社区分享的信息，帮助我们实现这些改动。

## 数据包版本 109.0

### 数据组件

#### 新增 `minecraft:block_transformer`

- 该组件允许物品在玩家交互时将方块转换为另一种方块。
- 格式：对象列表，字段如下：
  - `block_state_provider` - 方块状态提供器，用于提供要变换为的方块状态；
    - 若该方块状态提供器未返回结果（例如 `rule_based_state_provider`），则将尝试列表中的下一个规则；
    - 可用的方块状态提供器集合与世界生成中使用的相同。
  - `sound` - 可选字段，交互时播放的音效事件（例如 `minecraft:item.axe.strip`）；未指定则默认为无音效。
  - `particle` - 可选字段，交互时播放的粒子效果；未指定则默认为 `none`（可选值：`none`、`scrape`、`wax_on`、`wax_off`）。
  - `disallowed_faces` - 可选字段，指出在被点击方块上哪些面不能进行交互；若交互面为被禁止的面，则尝试下一个规则；缺省为空列表。可选值：`up`、`down`、`north`、`south`、`east`、`west`。
  - `loot` - 可选战利品表，指定成功转换时掉落物品的战利品表，例如 `minecraft:till/rooted_dirt`；未指定则不使用战利品表。
  - `drop_strategy` - 可选字段，指定战利品从方块哪个位置掉落；未指定默认为 `from_middle`。可选值：`from_middle`（从方块中间）、`clicked_face`（从交互的面）。
  - `transform_type` - 可选字段，配置周围方块如何受转换影响；未指定默认为 `single_block`。可选值：`single_block`（仅影响被交互的方块）、`copper_chest`（当输入与输出均为任意风化状态的铜箱子时，转换会影响双箱子的两边）。
  - `consume_on_use` - 可选布尔值，决定该物品是否在使用后被消耗，仅适用于可堆叠物品；未指定默认为 `true`。
  - `item_damage_per_use` - 可选整数，决定非堆叠物品每次使用受到的耐久损耗；未指定默认为 `1`。

### 世界生成

#### 地物

- 新增 `minecraft:copy_properties_provider` 方块状态提供器：从被评估位置的方块复制常见方块状态属性到目标状态。格式为对象，字段：
  - `block_state_provider`：方块状态提供器，提供要在其上复制属性的方块状态；仅会复制源和目标方块共有的属性。

- `minecraft:stepped_column_cluster`（原名 `minecraft:basalt_columns`）已重命名并增强，提供更多控制：
  - 新增字段 `block`（要放置的方块状态提供器）；
  - 新增字段 `can_replace`（允许被柱子替换的方块谓词）；
  - 新增字段 `continue_through`（允许被视为柱子一部分的预先存在方块谓词）；
  - 新增字段 `cannot_place_on`（方块 ID 列表或带 # 的方块标签，表示开始放置时要避免的方块）；
  - 字段 `reach` 重命名为 `column_reach`；
  - 新增 `column_count`：int 提供器（1 到 150），生成柱子的数量；
  - 新增 `cluster_reach`：int 提供器（0 到 13），确定生成柱子的方形区域大小（还受 `height` 限制）。

- `minecraft:single_block_pillar`（原名 `minecraft:basalt_pillar`）已重命名并增强：
  - 新增字段 `block`、`can_replace`；
  - 新增 `direction`：竖直方向，取值 `up` 或 `down`；
  - 新增 `chance_to_continue`：0 到 1 的概率（默认 1），决定柱子是否继续延伸一个方块；
  - 新增 `cap_feature`：可选的放置地物，置于柱顶。

- `minecraft:random_neighbor_spread`（原名 `minecraft:glowstone_blob`）已重命名并增强：在以原点为起点的区域内随机放置方块，使每个放置的方块恰好有一个被接受的邻居。格式上新增字段 `block`、`accepted_neighbors`、`can_replace`、`attempts`（1 到 3000）、`xz_offset`（-16 到 16）与 `y_offset`（-16 到 16）。

- 新增 `minecraft:overlay` 地物类型：在同一位置放置一组地物。与 `minecraft:sequence` 不同，`overlay` 中的所有地物都会被放置，互不依赖放置成功与否。字段：`features`（非空的 Placed Feature ID 列表或 # 前缀的 Placed Feature 标签）。

- 新增 `minecraft:projected_random_patchy_square` 地物类型：生成一个方形区域，其中每个方块是否放置由其与中心的距离决定，中心更容易生成，放置的方块可以在满足条件时向下投影。字段包括 `block`、`project_through`（被视为“空”的方块谓词）、`size`（1 到 16）和 `max_projection_height`（最大向下投影高度）。

#### 放置修饰器

- 将 `random_offset` 重命名为 `minecraft:offset`：
  - 移除字段 `xz_spread` 与 `y_spread`；
  - 新增字段 `x`、`y`、`z`：轴向偏移的整数提供器，范围 -16 到 16。

#### 噪声设置

- `spawn_target` 字段已更新，允许使用任意密度函数，而不限于 `noise_router` 中定义的子集。
- 候选生成位置的适宜度通过在任意 n 维空间中计算到一组目标的平方欧氏距离来确定。例如，一条轴可能是 `temperature`，另一条轴是 `erosion`，适宜度由距某个矩形区域（例如 `temperature=0.1..0.3`、`erosion=0.2..0.5`）的距离决定，越接近该区域的候选位置越优先。
- 格式：目标超立方体列表。目标格式为：将密度函数映射到该轴上的一个范围，范围可以是单个浮点数、两个浮点数组 `[min, max]`，或包含 `min` 与 `max` 字段的对象。

#### 方块谓词

- 新增 `minecraft:height_range`：用于将位置的高度与给定范围进行检查，格式：
  - `min_inclusive`：Vertical Anchor，表示最小高度（含）；
  - `max_inclusive`：Vertical Anchor，表示最大高度（含）。

#### 垂直锚点

- 为垂直锚点新增了一个选项 `relative_to_sea_level`，用于指定相对于维度海平面的偏移。

#### 雕刻器

- `worldgen/configured_carver` 注册表已重命名为 `worldgen/carver`。

- 更新了 `minecraft:cave` 雕刻器类型：原先在 `config` 字段下的所有字段已提升到顶层，新增/修改项包括：
  - 新增 `count`：非负整数提供器，指定从种子区块生成的洞穴隧道数量；
  - `yScale` 重命名为 `room_vertical_radius_multiplier`；
  - 新增可选 `start_vertical_radius_multiplier`：Float 提供器，用于第一个隧道段的竖向半径乘数，默认 `1.0`；
  - 新增 `thickness`：非负 Float 提供器，作为挖掘隧道半径的乘数；
  - 新增可选 `weird_thickness_bias`：布尔值；若为 `true`，则有 10% 概率对厚度乘以 `random(0,1) * random(0,3) + 1`；默认 `false`；
  - 移除 `replaceable` 字段（雕刻器将替换任何方块，地物除外）；
  - 移除 `lava_level` 字段（流体现在由维度的 `aquifer` 或 `sea_level` 配置选取）；
  - 移除 `debug_settings` 字段。

- 更新了 `minecraft:canyon` 雕刻器类型：同样将 `config` 下字段提升到顶层，并将 `yScale` 重命名并移动到 `shape.y_scale`；移除了 `replaceable`、`lava_level` 与 `debug_settings` 字段。

- 移除了 `minecraft:nether_cave` 雕刻器类型，改由 `minecraft:cave` 取代。

### 着色器与后处理

> [!DEVNOTE] 尽管资源包可以覆盖核心着色器，但覆盖核心着色器被视为不受支持且并非资源包的预期特性。这些着色器属于游戏内部实现的一部分，游戏内部实现可能随时更改。我们理解覆盖核心着色器可以实现一些很酷的资源包效果，许多效果暂无受支持的替代方案。未来我们希望提供更好且受支持的替代方式。

#### 与 OIT 相关的着色器更改

- 新增用于支持与顺序无关透明度的着色器：
  - `core/oit_composite.fsh`
  - `include/oit.glsl`
  - `include/oit_add_transmittance.glsl`
  - `include/oit_common.glsl`
  - `include/oit_depth_bounds.glsl`
  - `include/oit_depth_sample.glsl`
  - `include/oit_sample.glsl`
- 将 `core/terrain.fsh` 中的纹理采样逻辑抽取为 `include/texture_sampling.glsl`。
- 重命名了若干核心着色器：
  - `core/rendertype_clouds.vsh` -> `core/clouds.vsh`
  - `core/rendertype_clouds.fsh` -> `core/clouds.fsh`
  - `core/rendertype_world_border.vsh` -> `core/world_border.vsh`
  - `core/rendertype_world_border.fsh` -> `core/world_border.fsh`
- 更新核心着色器以支持 OIT，新增了若干预处理定义（defines）：
  - `OIT`：布尔值，表示是否启用 OIT；
  - `OIT_DEPTH_BOUNDS`：布尔值，表示是否在执行深度范围通道；
  - `OIT_TRANSMITTANCE`：布尔值，表示是否在执行透射率通道；
  - `OIT_ACCUMULATE`：布尔值，表示是否在执行累积通道；
  - `OIT_ALPHA_ONLY`：布尔值，表示是否正在执行深度范围或透射率通道中的任一项；
  - `OIT_ADDITIVE`：布尔值，指示 OIT 算法是否模拟加性混合；
  - `OIT_OPAQUE_PARTS_THRESHOLD`：浮点数，判断片元高于该 alpha 阈值是否被视为不通过 OIT 的不透明部分；
  - `WAVELET_RANK`、`COEFF_COUNT`、`COEFF_ATTACHMENT_COUNT`：OIT 算法内部使用的整数；
  - `B3D_IS_ZERO_TO_ONE`：布尔值，指示当前渲染后端是否使用深度范围 [0,1]（该宏对所有着色器全局有效，与 OIT 设置无关）。
- 受影响的着色器列表包括但不限于：`core/block.fsh`、`core/block.vsh`、`core/clouds.fsh`、`core/clouds.vsh`、`core/entity.fsh`、`core/entity.vsh`、`core/item.fsh`、`core/item.vsh`、`core/particle.fsh`、`core/position_color.fsh`、`core/terrain.fsh`、`core/terrain.vsh`、`core/text.fsh` 等。
- 移除了旧的透明度后期处理链：`post_effect/transparency.json` 与 `shaders/post/transparency.fsh`。
