## 新特性

### 斑驳森林生物群系

- 增加了一种新的森林类生物群系，生成在寒冷生物群系附近
- 会自然生成杨树和倒下的杨树
- 会生成少量云杉、红灌木、棕色蘑菇和枯叶堆
- 可以找到 Shelf Mushroom，其会生成在杨树和倒下的杨树树干上
- 斑驳森林里会生成寒带变种的绵羊、鸡、牛、猪以及狐狸和兔子
- 可以找到废弃营地的杨树森林变种

### 杨树

- 增加了一种新的树木，分为红色、橙色和黄色三种树叶变种
- 每种变种都有对应颜色的落叶粒子
- 杨树树叶会掉落杨树树苗，树苗可以随机长成三种变种中的任意一种
- 可以通过与流浪商人交易获得杨树树苗
- 杨树原木可以合成为杨树木头
- 杨树、杨树木头、去皮杨树原木和去皮杨树木头都有对应的木制方块、告示牌、门、活板门、船和运输船

### Shelf Mushroom

- 增加了一种新的蘑菇，会生成在杨树和倒下的杨树上
- 分为小型和大型两种尺寸
- 小型 Shelf Mushroom 可以使用骨粉长成大型 Shelf Mushroom
- 大型 Shelf Mushroom 被破坏后会掉落两个 Shelf Mushroom 物品
- 可以像其他蘑菇一样用于合成蘑菇煲
- 可以通过与流浪商人交易获得
- 有一点弹性，效果类似床
- 当生物在上面弹跳时，会发出弹跳音效

### 红灌木

- 增加了一种装饰性植物方块，会在斑驳森林中成片生成
- 可以用于堆肥桶

### 废弃营地结构

- 增加了一种新的结构，可以在下列生物群系中生成：
  - 草甸
  - 樱花树林
  - 森林
  - 桦木森林
  - 原始桦木森林
  - 繁花森林
  - 风袭森林
  - 斑驳森林
  - 针叶林
  - 积雪针叶林
  - 原始云杉针叶林
  - 原始松木针叶林
  - 稀疏丛林
  - 竹林
  - 沼泽
  - 热带草原
  - 疏林恶地
  - 苍白之园
- 不同生物群系会生成各自的结构变种
- 结构的一部分部件是通用的，所有变种都会使用
- 可以找到带战利品的箱子、铜箱子和木桶

### 羊毛楼梯和羊毛台阶

- 为每种颜色的羊毛方块都增加了对应的楼梯和台阶变种
- 羊毛楼梯和羊毛台阶和羊毛相同，当踩踏、丢弃或放置时都不会被幽匿感测体和监守者检测到

### 命令

#### `give`

- 当给予的物品数量超过上限时，`give` 现在会直接报错，而不再打印错误信息后返回 `0`

#### `tellraw`

- 原始聊天消息中的 `nbt` 组件最多只会解析 64000 次

#### `tick`

- 下列子命令现在在失败时会直接报错，而不再打印错误信息后返回 `0`
  - `tick step`
  - `tick step stop`
  - `tick sprint stop`

#### `team`

- `team join` 和 `team leave` 现在会返回实际发生变化的实体数量，不再把没有变更的实体算进去

#### `item`

- 先前使用槽位的地方现在改为接受槽位源
  - 槽位源会通过 `minecraft:command_slot_source` 解析
  - 可以以内联形式指定，也可以引用外部槽位源
  - 可以指向多个槽位
  - `replace` 的语义变化：
    - 槽位源里的物品会按顺序一个一个替换目标槽位里的内容
    - 目标槽位比来源物品多时，多余的槽位会被忽略
      - 例如，用 `(ABC)` 替换 `[12345]` 会得到 `[ABC45]`
    - 目标槽位比来源物品少时，多余的物品会被忽略
      - 例如，用 `(ABCDE)` 替换 `[123]` 会得到 `[ABC]`
  - 字符串形式的槽位范围写法 `<slot_type>` 或 `<slot_type>.<slot_number>` 仍然可用
    - 可以是单个槽位，例如 `armor.chest`
    - 也可以是多个槽位，例如 `container.*`
    - 其实际为 `minecraft:slot_range` 槽位源的简写
      - 例如，`hotbar.4` 与 `{type:"minecraft:slot_range",slots:"hotbar.4"}` 相同
- `from entity` 的物品来源现在可以使用选中多个实体的选择器
  - 多个实体的槽位会按顺序串联起来
- 增加了 `fill` 和 `override` 子命令，语法和 `/item replace` 一样
  - `/item fill`
    - 用来源物品依次填充每个目标槽位
    - 目标槽位比来源物品多时，会重复来源物品直到填满所有槽位
      - 例如，用 `(ABC)` 填充 `[12345]` 会得到 `[ABCAB]`
  - `/item override`
    - 用来源物品覆盖每个目标槽位中的内容
    - 目标槽位比来源物品多时，多余的槽位会被清空
      - 例如，用 `(ABC)` 覆盖 `[12345]` 会得到 `[ABC__]`，其中 `_` 表示空槽位

#### `execute`

- 增加了 `slots` 条件子命令
  - 用来检测方块或实体是否具有和给定槽位源匹配的槽位
  - 槽位源会通过 `minecraft:command_slot_source` 解析
  - 语法：
    - `/execute if|unless slots block <pos> <slot source>`
    - `/execute if|unless slots entity <targets> <slot source>`
- `items` 条件子命令现在也改为接受槽位源，而不是槽位
  - 槽位源会通过 `minecraft:command_slot_source` 解析
  - 语法：
    - `/execute if|unless items block <pos> <slot source> <item predicate>`
    - `/execute if|unless items entity <targets> <slot source> <item predicate>`

### 数据组件

#### `minecraft:provides_pottery_pattern`

- 增加了该组件，以记录物品携带的陶片图案
- 格式：一个命名空间 ID，指向 `minecraft:decorated_pot_pattern` 注册表中的条目

#### `minecraft:pot_decorations`

- 现在不再是 4 个物品 ID 的列表，而是一个带 4 个可选字段的对象
  - `back`
  - `left`
  - `right`
  - `front`

#### `minecraft:trim_material`

- 更改了内联材料定义的格式

### 纹饰材料

- 更改了 `trim_material` 注册表条目的格式：
  - `asset_name` 被替换为 `palette`，用于指定这份材料对应的调色板纹理
  - `override_armor_assets` 被移除，覆盖现在改由资源包里的装备资源定义
  - 具体格式请参阅资源包部分的调色板纹理说明

### 乐器

- 更改了 `instrument` 注册表条目的格式：
  - 增加了 `durability_damage` 字段，表示每次吹奏消耗多少耐久值
  - `use_duration` 现在可以为 `0`，表示没有吹奏冷却

### 方块实体数据

#### `minecraft:decorated_pot`

- `sherds` 字段不再存 4 个物品 ID 的列表，而是改成一个包含 4 个可选字段的对象
  - `back`
  - `left`
  - `right`
  - `front`

### 槽位源

- 槽位源现在可以放在数据包的 `slot_source` 文件夹里定义
  - 可以给接受槽位源参数的命令使用
  - 也可以在内联槽位源里用 `minecraft:reference` 引用
- `minecraft:contents` 现在也可以选中空槽位了

#### `minecraft:reference`

- 这个类型会直接使用所引用的槽位源
- 字段：
  - `name`：槽位源的命名空间 ID

#### `minecraft:slot_range`

- `source` 字段新增了 `container` 取值，其会从战利品上下文里提供的容器或物品栏中取槽位
- `source` 现在变成可选字段了
  - 缺省时默认为 `container`

### 战利品函数

- 增加了 `minecraft:command_slot_source` 战利品类型
  - 主要给 `/item` 和 `/execute` 这些命令解析槽位源使用
  - 需要以下参数：
    - `container`：正在被解析的容器或物品栏
    - `origin`：命令执行的位置
    - `this_entity`：命令执行者实体，可选

### 世界生成

#### 地物

- `worldgen/configured_feature` 注册表移到了 `worldgen/feature`，且配置现在直接写在根对象里，不再单独放进 `config` 字段

- 增加了 `minecraft:end_podium` 地物类型
  - 用于放置末地传送门底座
  - 现在会通过两个地物实例来在游戏事件中放置：
    - `end_podium_active`
    - `end_podium_inactive`
  - 字段：
    - `active`：表示传送门是否已激活

- `minecraft:tree` 增加了 `shelf_mushroom` 树木装饰器
  - 会以指定概率在原木上放置 Shelf Mushroom
  - 格式：
    - `probability`：0.0 到 1.0 之间的浮点数，表示尝试放置的概率

- 增加了 `poplar_foliage_placer` 树叶放置器
  - 会沿两种可能方向之一，以菱形放置杨树树叶
  - 格式：
    - `height`：5 到 16 之间的整数提供器，表示树叶高度
    - `side_hole_chance`：0.0 到 1.0 之间的浮点数，表示边缘树叶被随机移除的概率

- 增加了 `poplar_trunk_placer` 树干放置器
  - 会竖直放置杨树原木，并在树冠开始处放出树枝
  - 格式：
    - `trunk_height_above_branches`：0 到 8 之间的整数提供器，表示树枝上方的树干高度
    - `branch_amount`：1 到 4 之间的整数提供器，表示要生成多少根树枝

#### 噪声设置

- `surface_rule` 字段重命名为 `material_rule`
  - 现在可以直接写材料规则，也可以写材料规则引用

#### 材料规则注册表

- 新增了一个数据驱动注册表，读取自 `data/<namespace>/worldgen/material_rule/<id>.json`
- 其中的条目用于定义在世界某个位置要放什么基础方块
- 格式与先前在噪声设置里的 `surface_rule` 完全相同，仅增加了两个扩展：
  - 材料规则现在可以以内联形式写，也可以引用 `worldgen/material_rule` 里的条目
  - 材料条件现在可以引用 `worldgen/material_condition` 里的条目

#### 材料条件注册表

- 新增了一个数据驱动注册表，读取自 `data/<namespace>/worldgen/material_condition/<id>.json`
- 其中条目定义了材料规则里可以引用的条件，比如测试生物群系或地表厚度
- 格式和以前写在 `condition` 材料规则类型里的内容一样，只是材料条件现在也可以引用注册表条目

#### 结构放置

- 增加了 `minecraft:dimension_origin` 结构放置类型
  - 会在维度原点处放置结构的一个实例
  - 对于使用噪声设置的维度，原点由 `spawn_target` 决定，否则就是区块 `(0, 0)`
  - 无额外字段

#### 规则测试

- 这些规则测试会被 `ore` 地物和 `rule` 结构处理器使用

- 增加了 `minecraft:height_match`
  - 用来检查方块的 Y 坐标，只有在区间 `[min_inclusive, max_inclusive]` 内才返回 `true`
  - 字段：
    - `min_inclusive`：允许的最低 Y 坐标
    - `max_inclusive`：允许的最高 Y 坐标

- 增加了 `minecraft:all_of`
  - 用来检查同一个方块是否同时满足规则列表里的所有条件
  - 字段：
    - `rules`：要测试的规则列表
