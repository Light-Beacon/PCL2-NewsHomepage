## 已知问题
- Windows 上的独占全屏模式在某些情况下可能导致游戏崩溃，尤其在使用多台显示器时
- 使用 Wayland 进入独占全屏模式会导致游戏崩溃

## 新特性
- 旁观模式下的玩家现在可以与传送门交互来进行传送

## 更改

### 方块、物品和实体的细微调整
- 犰狳在浸入流体时将不再试图蜷缩

### 用户界面
- 移除了“原始鼠标输入”设置
- 在游戏内游玩时，鼠标输入现在始终使用相对鼠标移动模式
- 键位绑定现在使用物理按键，而非特定键盘布局的键码
- 无边框全屏现为默认的全屏模式
- 在无边框和独占全屏之间切换不再需要重启游戏
- macOS 不再支持独占全屏模式
- 最小窗口尺寸现在为 320×240 像素
- 在 macOS 上，输入文本时长按按键现在会显示原生的重音符号和候选窗口
- 在 Linux 上，如果 Wayland 可用，游戏现在将默认使用并优先选择它

#### 调试界面
- 调试界面现在支持与游戏其他部分不同的 GUI 尺寸
  - 这可通过按下`F
3 + F6`进入“调试选项”屏幕来进行自定义
  - 默认选项为“Auto”，会尝试使用比正常更大的尺寸
  - 另一个选项是“Unchanged”，与常规 GUI 尺寸匹配
  - 其余选项与原设置项的含义相同
- 添加了“player_speed”调试条目，以每刻方块数显示玩家的速度
- 调试界面现在会显示显示器的刷新率

### 创造模式物品栏
- 重新排序了矿物物品和方块的顺序，将非分级材料排在前面，然后是可合成装备的分级材料排在最后
  - 材料
    - 无材料等级的矿物
    - 未烧炼的矿物
    - 烧炼后的矿物
      - 粒
      - 锭
  - 建筑方块
    - 无材料等级矿物块及其变种
    - 烧炼后的矿物块及其变种
    - 铜制品系列
  - 由于铜类包含大量内容，其仍被置于建筑方块顺序的末尾
- 改进了“自然方块”标签页的排序，使所有组内内容的内部排序从主世界 -> 下界 -> 末地依次排列，来与其他标签页保持一致

### 环境属性

#### 新增 `minecraft:gameplay/natural_mob_spawns`
- 在环境属性源中定义生物生成
- 在世界生成放置期间，只有维度和生物群系会应用此环境属性
- 格式：包含以下字段的对象：
  - `spawns_by_category` - 生成类别到加权生成数据列表的映射
    - 生成数据格式：包含以下字段的对象：
      - `type` - 实体类型，要生成的实体
      - `count` - 整数提供器，生成的数量
  - `spawn_costs` - 实体类型到包含以下字段的对象的映射：
    - `energy_budget` - 浮点数，每次生成允许的能量变化
    - `charge` - 浮点数，现有生物会如何吸引或排斥其他带电生物
- 可用的属性修饰符：
    - `overlay`：对于每个生物类别，用高层的生成设置覆盖低层的，除非该类别未被包含。合并各层的生成消耗，如果两者定义了相同的实体类型，则用高层的生成消耗覆盖低层的
- 默认值：空

#### 新增 `minecraft:gameplay/creature_world_gen_spawn_probability`
- 设置在世界生成期间，为生物类别中定义的生物运行一次生成迭代的概率
- 只有维度和生物群系会应用此环境属性
- 格式：[0,1) 范围内的浮点数
- 默认值：0.1

#### 更改了 `minecraft:visual/ambient_particles`
- 现在支持时间轴关键帧之间的插值（概率将进行交叉淡化处理）
- 引入了对新修饰符 `append` 的支持。与完全替换粒子列表的 `override` 不同，此修饰符会将所有元素与下层进行连接

### 数据组件

#### 新增 `minecraft:cooking_fuel`
- 描述了一种可用作熔炉、烟熏炉或高炉燃料的物品
- 格式：包含以下字段的对象
  - `burn_time` - 指向 `minecraft:number_provider` 注册表元素的命名空间 ID，表示该燃料的燃烧时间（以刻为单位）
  - `speed_multiplier` - 指向 `minecraft:number_provider` 注册表元素的命名空间 ID，表示烹饪/熔炼的速度

#### 新增 `minecraft:brewing_fuel`
- 描述了一种可用作酿造台燃料的物品
- 格式：包含以下字段的对象
  - `uses` - 指向 `minecraft:number_provider` 注册表元素的命名空间 ID，表示该燃料在被消耗前可以进行酿造的次数
  - `speed_multiplier` - 指向 `minecraft:number_provider` 注册表元素的命名空间 ID，表示酿造的速度
- 注意：`#brewing_fuel` 物品标签已被移除，不再用于注册新的酿造燃料

#### 新增 `minecraft:sign_text_front` 和 `minecraft:sign_text_back`
- 两个相同的组件，分别表示告示牌正面和背面的文字
- 内容将显示在物品提示框中
- 格式与 `minecraft:sign` 和 `minecraft:hanging_sign` 方块实体上的 `front_text` 和 `back_text` 字段相同，即：
  - `messages` - 文本组件列表
  - `filtered_messages` - 可选的文本组件列表，如果省略，将设置为与 `messages` 相同的值
    - `messages` 和 `filtered_messages` 的条目数必须相同
  - `color` - 可选染料颜色，默认：`black`
  - `has_glowing_text` - 可选布尔值，默认：`false`

#### 新增 `minecraft:waxed`
- 用于标记内容已被上蜡的方块
- 无字段

#### 新增 `minecraft:cushion/color`
- 当坐垫使用物品放置时，应用于坐垫实体
- 格式：16 种染料颜色之一

#### 新增 `minecraft:villager_food`
- 代表村民可以食用的物品。村民会拾取带有此组件的物品，以及 `#villager_picks_up` 标签中的物品
- 格式：包含以下字段的对象
  - `nutrition` - 正整数，表示该物品被村民食用后恢复的饥饿值

#### 新增 `minecraft:mob_visibility`
- 代表带有适当 `minecraft:equippable` 组件的物品所提供的可见度百分比，用于修改生物能够检测到实体的范围
- 格式：包含以下字段的对象
  - `targeting_entity` - 实体 ID、命名空间实体 ID 列表或带哈希前缀的实体标签，用于匹配
  - `visibility` - 浮点数（介于 0.0 和 10.0 之间），0.0 将生物检测实体的范围减小到 2 格，而 10.0 则将检测范围扩大十倍
  - 虽然多个带有此组件的物品效果可以叠加，但最大视野范围仍不会超过 10.0

  - 将 `cooking_total_time` 字段从短整型改为整型，表示当前烹饪过程所需的总时间
  - 将 `lit_time_remaining` 字段从短整型改为整型，表示剩余的燃烧时间
  - 将 `lit_total_time` 字段从短整型改为整型，表示上次补充燃料时添加的总燃烧时间
