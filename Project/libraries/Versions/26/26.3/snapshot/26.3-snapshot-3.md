## 新特性

- 增加了 `坐垫` 物品
- 增加了 `麦秆床`
- 增加了对哥特语（Gothic）语的支持

### 坐垫

- 坐垫是一种物品，玩家可以将其放置在世界中并与之交互以坐下
- 可制作为 16 种颜色的变体
- 坐垫的合成配方使用 3 个相同颜色的羊毛台阶

#### 坐垫 放置规则

- 坐垫可放置在任意平坦表面上
- 坐垫在水平方向上始终与网格对齐，但在垂直方向上位于支撑表面之上
- 坐垫不可移动
- 当支撑方块被移除时，坐垫会破坏
- 坐垫不具有碰撞体，可与其他对象重叠（但不能与其他坐垫重叠）

### 麦秆床

- 麦秆床是一种新的床类型，可在夜间睡觉而不设定出生点
- 麦秆床仅可使用一次，使用后会被摧毁
- 麦秆床不能在下界或末地使用，尝试在这些维度睡觉时床会被摧毁
- 合成配方使用 3 个干草捆，合成将产出 4 个麦秆床
- 统计界面新增了一个统计项，用于记录在麦秆床中睡觉的次数

## 技术性更改

- 数据包版本现为 `110.0`
- 资源包版本现为 `91.0`

## 数据包版本 110.0

### 命令

#### 新增 `posteffect` 命令

- 允许向玩家添加或移除后效果
- 这些是应用于玩家屏幕的后处理着色器，会在游戏完成常规渲染后执行
- 后效果存在于客户端（资源包中），服务器无法确定某个后效果是否会实际应用
- 后效果本身以前已经是资源包格式的一部分，但之前只能使用少数硬编码的效果

> [!DEVNOTE] 这使得资源包中的自定义着色器可以被完整使用，但请注意着色器仍可能随时更改，相关功能（命令、后效果或着色器）未来可能被重新设计

用法示例：

- `/posteffect add <player> <posteffect>` — 将后效果添加到玩家（若未存在）
- `/posteffect clear <player>` — 从玩家处移除所有后效果（若存在）
- `/posteffect list <target>` — 列出应用于玩家的所有后效果（若存在）
- `/posteffect remove <player> <posteffect>` — 从玩家处移除指定后效果（若存在）

#### 对 `place` 子命令的更改

- `feature` 子命令现在允许内联定义要放置的地物

### 环境属性

#### 修改 `minecraft:gameplay/bed_rule`

- 字段 `explodes` 已重命名为 `destroy_on_use`
- 新增 `destroy_on_leave` 可选布尔值，若为 `true`，离开床后该床会被摧毁；若省略，则默认为 `false`

#### 新增 `minecraft:gameplay/straw_bed_rule`

- 与 `minecraft:gameplay/bed_rule` 格式相同，但用于麦秆床的规则

### 数据组件

#### 更新 `minecraft:block_transformer`

- 新增可选布尔字段 `update_from_neighbors`，用于控制被转换的方块是否应根据邻居方块发生更新
  - 默认值为 `true`

#### 新增 `minecraft:compostable`

- 表示物品可作为堆肥材料使用以及其向堆肥桶添加的层数
- 格式：对象，字段包括：
  - `layers` — 命名空间引用，指向 `minecraft:number_provider` 注册表中的元素

### 战利品表

- 多个方块战利品表现在使用谓词来检查工具是否具有精准采集或是否为剪刀，而不是内联复制这些检查逻辑

### 谓词

#### 新增原版谓词

以下项目已添加到此前为空的 `minecraft:predicate`：

- `minecraft:tool/can_silk_touch`
- `minecraft:tool/can_shear`

#### 药水相关谓词

- `minecraft:potion_contents` 数据组件谓词已更改，允许匹配特定的药水效果此前的谓词格式使用药水类型列表，新的格式允许更精细的匹配
- 下列字段可任意组合使用，均为可选：
  - `potions` — 要匹配的药水类型列表或药水标签
  - `effects` — 要匹配的 `mob_effects` 谓词的集合谓词，其格式为：
    - `contains` — 要匹配的一组 `mob_effects` 谓词列表；
    - `count` — 匹配特定数量的 mob effect 谓词，格式与 `minecraft:effects` 实体子谓词相同：
      - `test` — 要匹配的 `mob_effects` 谓词；
      - `count` — 所需匹配数，或使用 `min` 与 `max` 指定匹配数量范围；
    - `size` — 要求 mob effects 列表具有特定大小，或使用 `min` 与 `max` 指定范围
- `mob_effects` 谓词的格式为将效果类型映射到该效果的属性，所有字段均为可选，只有提供的字段才用于匹配（示例和具体 JSON 格式保持原样）

### 数值提供器

- 新增 `minecraft:number_provider` 注册表

#### 新增原版数值提供器

- `minecraft:compostable/low`：如果 `minecraft:block_state` 上下文参数为一个空的堆肥桶，则返回 1，否则有 30% 概率返回 1
- `minecraft:compostable/low_medium`：若为空堆肥桶返回 1，否则有 50% 概率返回 1
- `minecraft:compostable/medium`：若为空堆肥桶返回 1，否则有 65% 概率返回 1
- `minecraft:compostable/medium_high`：若为空堆肥桶返回 1，否则有 85% 概率返回 1
- `minecraft:compostable/always_add_one`：始终返回 1

#### 新增 `minecraft:number_dispatcher` 类型

- 返回第一个满足条件的条件，若无满足则返回 `default`
- 字段：
  - `cases` — 按顺序尝试的条件列表，每个条件包含：
    - `condition` — 一个 `minecraft:predicate`；
    - `number_provider` — 在条件满足时执行的 `minecraft:number_provider`；
  - `default` — 可选的 `minecraft:number_provider`，若省略则默认为常量 `0`

#### 新增 `minecraft:conditional_value` 类型

- 若条件满足返回 `on_true` 的值，否则返回 `on_false` 的值
- 字段：
  - `conditions` — 一个 `minecraft:predicate`；
  - `on_true` — 条件满足时执行的 `minecraft:number_provider`；
  - `on_false` — 可选的 `minecraft:number_provider`（省略时默认为常量 `0`）

#### 新增 `minecraft:weighted_list`

- 从加权的数值提供器分布中返回一个值
- 字段：
  - `distribution` — 对象列表，每项包含：
    - `data` — 一个 `minecraft:number_provider`；
    - `weight` — 正整数，权重

### 配方

#### 药水酿造配方

- 药水酿造配方现已数据驱动可以通过数据包添加新配方，也可以修改原版配方酿造配方使用 `minecraft:brewing` 类型
- 主要字段包括：
  - `input` — 酿造基底，必须包含 `item` 字段，可选使用 `minecraft:potion_contents` 数据组件谓词的 `potion_contents` 字段
  - `reagent` — 酿造原料，必须包含 `item` 字段，可选使用 `minecraft:potion_contents` 数据组件谓词的 `potion_contents` 字段
  - `output` — 物品堆叠，合成结果

以下是原版配方示例：

```
{
  "type": "minecraft:brewing",
  "input": {
    "item": "minecraft:potion",
    "potion_contents": {
      "potion": "minecraft:water"
    }
  },
  "output": {
    "components": {
      "minecraft:potion_contents": {
        "potion": "minecraft:awkward"
      }
    },
    "id": "minecraft:potion"
  },
  "reagent": {
    "item": "minecraft:nether_wart"
  }
}
```

三个槽位都可以放置任意物品，例如：

```
{
  "type": "minecraft:brewing",
  "input": { "item": "minecraft:bucket" },
  "output": { "id": "minecraft:sulfur_cube_bucket" },
  "reagent": { "item": "minecraft:potent_sulfur" }
}
```


### 世界生成

#### 地物

- 移除了下列地物类型：
  - `minecraft:coral_mushroom`
  - `minecraft:kelp`
  - `minecraft:seagrass`
  - `minecraft:sea_pickle`

> [!DEVNOTE] 这些特性类型不再需要，因为其先前由硬编码实现的特性现在可用其它更好配置的功能来实现

- 更改了 `minecraft:coral_claw` 与 `minecraft:coral_tree`：新增字段 `feature`，用于在每个目标方块位置放置指定的地物

#### 放置修饰器

- 新增 `minecraft:cuboid` 用于在长方体形状中重复放置特性

新增字段：
  - `xz_size`：1 到 16 的 Int 提供器，表示长方体在 x 与 z 轴方向的大小；
  - `y_size`：1 到 16 的 Int 提供器，表示长方体在 y 轴方向的大小；
  - `include_interior`：可选布尔（默认 `true`），是否放置长方体内部方块；
  - `include_edges`：可选布尔（默认 `true`），是否放置长方体边缘方块；将其设为 `false` 会生成更圆润的长方体外观

- 新增 `minecraft:random_chance`，用于在随机概率下决定是否放置特性

字段格式：
  - `chance`：0 到 1 的浮点数，表示放置特性的概率

#### 方块状态提供器

- 更新了 `minecraft:copy_properties_provider`：将字段 `source_block_state_provider` 重命名为 `source`

- 新增 `minecraft:random_block_provider`：为随机选择的方块返回默认方块状态
  - 格式：
    - `blocks`：方块 ID 列表或带 # 的方块标签，指定可随机选择的方块集合

- 修改了 `minecraft:rotated_block_provider`：现在可作用于任何带有 `axis` 或 `facing` 属性的方块
  - 字段 `state` 现为方块状态提供器；
  - 新增可选字段 `direction` 用于指定方向，若省略则随机选择方向

#### 规则测试

这些谓词用于 `ore` 地物（`targets[].target`）以及 `rule` 结构处理器（`rules[].input_predicate` 与 `rules[].location_predicate`）

- 新增 `minecraft:any_of`：检查一组 Rule Tests，若其中任一项对同一方块返回 `true` 则条件满足
  - 格式：对象，字段 `rules` 为 Rule Tests 列表
- 新增 `minecraft:not`：检查某个 Rule Test 对指定方块是否返回 `false`
  - 格式：对象，字段 `rule` 为要测试的 Rule Test

### 着色器与后处理效果

#### 始终生效的后效果支持

- 资源包现在可以定义 `minecraft:end_of_frame` 后效果
- 该效果会在其他后效果之前应用，包括由新的 `posteffect` 命令启用的那些效果
- 只要资源包被加载，该后效果始终开启，且无法由 `posteffect` 命令控制
- 与资源包资源的常规行为一致，如果多个资源包定义了此效果，则以列表中最后一个资源包的定义为准
