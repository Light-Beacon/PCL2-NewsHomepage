## 数据包版本 101

- "默认组件" 报告生成器不再为没有任何组件的实体输出文件

### 命令

#### 对 `time` 的更改

语法:

- `time [of <clock>] rate <rate>` - 设置该时钟使用的速率倍率
  - 注意：这只会改变世界时钟以及所有依赖该时钟的时间线推进其内部计时器的速率
    - 例如，对于 `minecraft:overworld` 时钟，更大的值会让昼夜循环过得更快，但实际游戏模拟速度不会提升（不同于 `/tick rate`）
  - `rate` 为 `1` 时表示正常速度
  - `rate` 是一个介于 `0`（不含） 和 `1000`（含）之间的浮点数

### 谓词

#### 新增 `minecraft:environment_attribute_check` 战利品表谓词

精确匹配给定位置上的某个环境属性值。注意：只要该环境属性会因位置不同而变化，此谓词就要求上下文中设置 `origin` 位置。

格式：具有以下字段的对象：

- `attribute` - 要检测的环境属性 ID
- `value` - 要精确匹配的环境属性值
- 例如：`{condition: 'environment_attribute_check', attribute: 'gameplay/piglins_zombify', value: true}`

### 数字提供器

#### 新增 `minecraft:environment_attribute`

获取并提供某个环境属性的值（前提是该值可以表示为数字）。注意：只要该环境属性会因位置不同而变化，此提供器就要求上下文中设置 `origin` 位置。

格式：具有以下字段的对象：

- `attribute` - 要获取的环境属性 ID
- 例如：`{type: 'environment_attribute', attribute: 'gameplay/sky_light_level'}`

### 世界生成

#### Flower 地物

- 由骨粉生成的地物现在不再局限于 `flower` 地物类型，而是改由配置地物标签 `#can_spawn_from_bone_meal` 控制
- `flower`、`flower_no_bonemeal` 与 `random_patch` 地物类型已被移除
  - 现在可改为使用一系列 `count` 与 `random_offset` 放置修饰器来表示这类簇状分布

#### 整数提供器

新增梯形整数提供器，可按梯形分布随机选取一个值，类似于 `trapezoid` 浮点数提供器。

格式：具有以下字段的对象：

- `min`：整数，生成值的最小值
- `max`：整数，生成值的最大值
- `plateau`：整数，分布中“平台”部分的宽度，在该范围内所有值出现的概率相同
  - 值为 `0` 时等价于三角分布
  - 值为 `max-min` 时等价于均匀分布

### 标签

#### 配置地物标签

- 新增 `#can_spawn_from_bone_meal` - 被加入某个生物群系后，可在该生物群系中使用骨粉生成的地物

### 文本组件

#### `minecraft:object`

- 新增一个名为 `fallback` 的可选字段，其中包含一个文本组件，当对象组件本身无法显示时使用（例如在服务器日志中打印消息或进行旁白时）
- `player` 类型的对象（玩家头颅）现在不能再用于服务器状态消息（Motd）中
  - 该类型的所有组件都会被替换为回退文本

## 资源包版本 84

- 更新了 Pup 的纹理，以修复纹理重叠的问题

### 物品模型

#### `minecraft:end_cube` 特殊模型类型

- 一种新的特殊模型类型，可渲染具有与末地传送门方块和末地折跃门方块相同纹理效果的立方体
- 字段：
  - `effect` - 要应用的纹理效果，可选值为：`portal`、`gateway`

### 方块状态渲染

- 末地折跃门和末地传送门现在在方块展示实体上渲染时，应与它们在世界中放置时的外观一致

### 着色器与后处理效果

- `core/rendertype_translucent_moving_block` 着色器已被移除，改用 `core/block`
