## 更改
- 重制了更多幼年生物的外观
- 幼年北极熊现在不再攻击狐狸
- 幼年美西螈不再播放死亡动画
- 修复了幼年鸡翅膀以及热带鸡的条纹

### 幼年生物

- 更新了下列生物的模型与纹理：
  - 蜜蜂
  - 狐狸
  - 山羊
  - 骆驼
  - 犰狳
  - 北极熊
  - 羊驼
- 在幼年骆驼上的鞍不再会被渲染
- 骆驼尸壳不再拥有幼年模型

### UI

- 随本次版本对世界数据格式的大幅更改，需升级的世界现在会显示“升级并打开”，而不是“进入选中的世界”；在世界升级完成之前，“编辑世界”与“重新创建世界”按钮将被禁用
  - 新增了一个界面用于显示升级进度

> [!DEVNOTE] 鉴于升级世界存在额外风险，在此快照周期中，首次加载你的世界时我们将强制进行备份。升级完成后，你将无法在旧版本中加载该世界。

## 数据包版本 99

### 世界存档

- 所有默认维度现在存储于 `dimensions` 子文件夹
  - 主世界的数据已从根目录移动到 `dimensions/minecraft/overworld`
    - 包括 `region`、`entities` 与 `poi` 文件夹，以及 `data` 中的相关文件
  - 下界的数据已从 `DIM-1` 移动到 `dimensions/minecraft/the_nether`
  - 末地的数据已从 `DIM1` 移动到 `dimensions/minecraft/the_end`
  - 顶层的 `data` 文件夹如今仅用于跨维度共享的数据，不再包含主世界的数据
- 玩家存储已移动到 `players` 子目录
  - `advancements` -> `players/advancements`
  - `playerdata` -> `players/data`
  - `stats` -> `players/stats`
- 世界资源包（`resources.zip`）已移动到 `resourcepacks` 子目录
- `data` 文件夹中保存的数据现已使用命名空间
  - 意味着所有该类数据将存放在相应 `data` 文件夹内的命名空间子目录中
    - 所有原版数据文件已据此移动
    - 例：`data/scoreboard.dat` -> `data/minecraft/scoreboard.dat`
  - 命令存储现改为在命名空间子目录中存放，而非以命名空间后缀命名
    - 例：`data/command_storage_foo.dat` -> `data/foo/command_storage.dat`
  - `chunks.dat` 重命名为 `chunk_tickets.dat`
  - 地图数据现在存放在 `data/minecraft/maps` 子目录中
    - 地图索引文件改为仅使用数字命名
      - 例：`map_1.dat` -> `1.dat`
    - `idcounts.dat` 重命名为 `last_id.dat`
  - 不再存在 `raids_end.dat` 的特殊情况
    - 末地同样使用 `raids.dat`
- 由结构方块保存的结构由 `generated/namespace/structures` 移动到 `generated/namespace/structure`

#### 对 `level.dat` 的更改

- `Player` 标签已被 `singleplayer_uuid` 标签取代，用于引用要使用的玩家数据文件
- 与难度相关的设置已移动到 `difficulty_settings` 标签
  - `difficulty`（原 `Difficulty`）现为字符串而非整数
    - 允许值：`peaceful`、`easy`、`normal`、`hard`
  - `locked`（原 `DifficultyLocked`）
- 末影龙战斗的相关数据已移至数据文件夹
  - 现可在 `dimensions/minecraft/the_end/data/minecraft/ender_dragon_fight.dat` 找到
  - 末影龙战斗可在其他维度中被激活，详见下方维度类型部分
  - 数据更改：
    - `needs_state_scanning`（原 `NeedsStateScanning`）
    - `dragon_killed`（原 `DragonKilled`）
    - `previously_killed`（原 `PreviouslyKilled`）
    - `respawn_stage`（取代 `IsRespawning`）
      - 允许值：`start`、`preparing_to_summon_pillars`、`summoning_pillars`、`summoning_dragon`、`end`
    - `dragon_uuid`（原 `Dragon`）
    - `exit_portal_location`（原 `ExitPortalLocation`）
    - `gateways`（原 `Gateways`）
    - 新增 `respawn_time` —— 自当前重生阶段开始以来已流逝的时间
    - 新增 `respawn_crystals` —— 触发重生流程所用末地水晶的 UUID 列表
- 流浪商人数据已移至数据文件夹
  - 现可在 `data/minecraft/wandering_trader.dat` 找到
  - 数据更改：移除 `WanderingTraderId`；`WanderingTraderSpawnChance` 重命名为 `spawn_chance`；`WanderingTraderSpawnDelay` 重命名为 `spawn_delay`
- `CustomBossEvents` 已移至数据文件夹（`data/minecraft/custom_boss_events.dat`）
- 天气相关数据已移至数据文件夹（`data/minecraft/weather.dat`）
  - 更名：`clear_weather_time`（原 `clearWeatherTime`）、`rain_time`（原 `rainTime`）、`thunder_time`（原 `thunderTime`）
- `ScheduledEvents` 已移至数据文件夹（`data/minecraft/scheduled_events.dat`）
  - 更名：`id`（原 `Name`）、`trigger_time`（原 `TriggerTime`）、`callback`（原 `Callback`）
    - 其内部字段更名：`type`（原 `Type`）、`id`（原 `Name`）
- `game_rules` 已移至数据文件夹（`data/minecraft/game_rules.dat`）
- `WorldGenSettings` 已移至数据文件夹（`data/minecraft/world_gen_settings.dat`）
  - 更名：`generate_structures`（原 `generate_features`）
- `world_clocks` 已移至数据文件夹（`data/minecraft/world_clocks.dat`）

### 世界生成

- 基于规则的方块状态提供器现已将 `fallback` 改为可选并允许为空
  - 若触发了空的 fallback，使用它的特征将不会放置任何内容
  - 当前使用基于规则的方块状态提供器的配置为 `disk` 特征中的 `state_provider`
- `forest_rock` 特征重命名为 `block_blob`
- `block_blob` 特征配置现支持以下参数：
  - `state`：岩石所用的方块
  - `can_place_on`：定义岩石可放置于哪些方块之上的方块谓词
- `ice_spike` 特征重命名为 `spike`
- `spike` 特征配置现支持以下参数：
  - `state`：尖（冰）刺所用的方块
  - `can_place_on`：定义尖刺可放置于哪些方块之上的方块谓词
  - `can_replace`：定义尖刺可替换哪些方块的方块谓词
- `huge_red_mushroom` 与 `huge_brown_mushroom` 特征配置新增参数：
  - `can_place_on`：定义巨型蘑菇可放置于哪些方块之上的方块谓词
- `alter_ground` 树木装饰器配置更改：`provider` 现为基于规则的方块状态提供器
- `tree` 特征配置用单个 `below_trunk_provider` 基于规则的方块状态提供器取代了 `force_dirt` 与 `dirt_provider` 参数：
  - 格式：
    - `fallback`：可选的方块状态提供器
    - `rules`：规则列表
      - `if_true`：在提供方块之前检查方块位置的方块谓词
      - `then`：方块状态提供器
  - 默认值：

```
"below_trunk_provider": {
    "rules": [
        {
            "if_true": {
                "type": "minecraft:not",
                "predicate": {
                    "type": "minecraft:matching_block_tag",
                    "tag": "minecraft:cannot_replace_below_tree_trunk"
                }
            },
            "then": {
                "type": "minecraft:simple_state_provider",
                "state": {
                    "Name": "minecraft:dirt"
                }
            }
        }
    ]
}
```

- 先前使用 `force_dirt` 的所有树木现在拥有一个始终提供 `dirt` 方块的 `below_trunk_provider`

### 标签

#### 方块标签

- 将 `#dirt` 拆分为多个方块标签
  - `#dirt` 现仅包含泥土、粗泥土与缠根泥土
  - `#mud` 包含泥与红树林泥根
  - `#moss_blocks` 包含苔藓方块与苍白苔藓方块
  - `#grass_blocks` 包含草方块、菌丝与灰化土
- 新增 `#substrate_overworld`，其包含 `#dirt`、`#mud`、`#moss_blocks` 与 `#grass_blocks`
  - 用作世界生成条件的集合
- 新增以下标签，用于定义可在适用的树或竹子下方被替换为灰化土的地面方块：
  - `#beneath_tree_podzol_replaceable`
  - `#beneath_bamboo_podzol_replaceable`
- 新增 `#cannot_replace_below_tree_trunk`，其包含 `#dirt`、`#mud`、`#moss_blocks` 与 `podzol`
- 新增以下标签，定义各自特征可放置于哪些方块之上：
  - `#forest_rock_can_place_on`
  - `#huge_brown_mushroom_can_place_on`
  - `#huge_red_mushroom_can_place_on`
- 新增 `#ice_spike_replaceable`，定义冰刺特征可替换的方块

#### 物品标签

- 新增以下标签，其内容与对应的方块标签镜像：
  - `#mud`
  - `#moss_blocks`
