## 数据包版本 97

- 新增方块标签
- 新增流体标签
- 调整了 `time` 命令的格式以适配“世界时钟”与“时间线”
- 更新了 `time_check` 条件以指定世界时钟

### 世界时钟

世界时钟包含一个每刻递增的内部时间。可在数据包的 `world_clock` 注册表下定义（位于 `data/<namespace>/world_clock/<id>.json`）。

- 世界时钟可以暂停或恢复，并可通过 `time` 命令修改或查询其时间
- 格式：无字段对象

### 时间线

- 新增 `clock` 字段：世界时钟 ID，指定该时间线绑定的世界时钟
  - 该字段为必填；为保持与旧行为一致，可使用 `minecraft:overworld` 时钟
- 新增可选 `time_markers` 字段：一个映射，键为命名空间的时间标记 ID，值为整数或“时间标记对象”
  - 格式：
    - 非负整数，表示该时间标记对应的刻数
    - 或对象，包含：
      - `ticks`：`0` 到 `period_ticks` 之间的整数，表示该时间标记对应的刻数
      - `show_in_commands`：可选布尔值，指定该时间标记是否在命令补全中显示
        - 默认：`false`

#### 时间标记

时间标记为特定世界时钟中的某个（可重复的）时间点指定一个名称。时间标记取代了先前 `/time set` 可用的预定义名称（如 `day`、`night`）。

时间标记存在于特定世界时钟的上下文中。这意味着尽管不同的时间标记由不同的时间线定义，但对于某个世界时钟，同一 ID 仅能存在一个时间标记；同时，两个不同的世界时钟可以拥有相同名称的时间标记。

#### 内部使用的时间标记

部分时间标记用于驱动原版玩法逻辑。通常它们被配置为不在 `/time` 命令中显示。

- `minecraft:wake_up_from_sleep` —— 玩家在该维度睡醒后，默认时钟前进到的时间点
- `minecraft:roll_village_siege` —— 在到该维度默认时钟的此时间点时，会检定僵尸围城事件是否发生

### 命令

#### 对 `swing` 的更改

- 现在两个参数都可缺省，并回调到如下默认值：
  - 实体选择器默认为 `@s`
  - 要挥动的手默认为 `mainhand`

#### 对 `time` 的更改

- `time` 命令现基于“世界时钟”运作：
  - 可使用 `time of <clock> ...` 显式指定要操作的时钟
  - 若未指定时钟（保持 `/time ...` 的旧用法），则使用该维度的默认时钟
- `set` 与 `add` 子命令的返回值（例如用于 `/execute store`）现在为该时钟的“已流逝总刻数”，而非“当前昼夜时间”

语法：

- `time [of <clock>] set <time>` —— 将该时钟的已流逝总刻数设置为指定值
- `time [of <clock>] set <timemarker>` —— 将该时钟推进到下一个指定时间标记出现的时刻
  - 对于 `minecraft:overworld` 时钟，可用的时间标记有：`day`、`noon`、`night`、`midnight`
- `time [of <clock>] add <time>` —— 为该时钟增加（或减少）刻数，允许负数
- `time [of <clock>] pause` —— 暂停该时钟
- `time [of <clock>] resume` —— 恢复该时钟
- `time [of <clock>] query <timeline>` —— 显示并返回该时间线周期内的当前刻数
- `time [of <clock>] query <timeline> repetitions` —— 显示并返回该时间线已重复的次数
- `time [of <clock>] query time` —— 显示并返回该时钟的已流逝总刻数
- `time query gametime` —— 与以往一致，返回世界层面的已流逝总刻数（不受 `/time` 修改时钟的影响）

### 游戏测试环境

#### `clock_time` 替代 `time_of_day`

- 新增必填字段 `clock`：世界时钟 ID，用于指定设置时间所依据的时钟（替代原有 `time` 字段）

### 谓词

#### `minecraft:time_check` 战利品表谓词

- 新增 `clock` 字段：世界时钟 ID，指定进行时间检查所使用的世界时钟

### 配方

- `result` 字段在所有包含它的配方类型中现在更为一致：
  - 现在接受简写形式（如 `"minecraft:foo"`，等价于 `{"id":"minecraft:foo", "count": 1}`）
  - 对 `smelting`、`blasting`、`smoking`、`campfire_cooking` 等配方类型，该字段现在也接受 `count`

### 世界生成

#### 维度类型

- 新增字段：
  - `default_clock`：可选，世界时钟 ID
    - 指定用于 `time` 命令的默认时钟
      - 若未指定，`time` 命令将需要显式提供时钟参数
    - 同时指定 `minecraft:wake_up_from_sleep` 与 `minecraft:roll_village_siege` 时间标记所作用的时钟
      - 若未指定，这些时间标记在该维度中将永不生效
