## 新特性

- 更新了进度以包含 Camel Husk 与 Parched
- 新增图形选项“各向异性过滤”

### Parched

- 新的骷髅变种
- 在沙漠 0 光照等级生成，替换部分普通骷髅
- 在阳光下不会燃烧
- 射出虚弱之箭
- 射速慢于普通骷髅，类似沼骸
- 不会在洞穴生成，遵循尸壳的生成规则

### Camel Husk

- 新的骆驼亡灵生物变种
- 在沙漠 0 光照等级生成
- 阳光下不燃烧
- 生成时身上骑乘两个生物：一个手持矛的尸壳和一个 Parched
- 不会在洞穴生成，遵循尸壳的生成规则
- 若骑手为敌对生物则为敌对
- 无骑手时为被动
- 像其他敌对生物一样会消失，但被玩家骑乘时会变为持久存在
- 最爱食物为兔子脚
- 不能繁殖
- 作为坐骑，其行为与普通骆驼类似

### 矛武器

#### 突进魔咒

- 降低了所有等级突进的饥饿值消耗
  - 1 级消耗 1 点饥饿，2 级消耗 2 点，3 级消耗 3 点（此前为 4/5/6）
- 为所有等级的突进新增 1 点耐久度损耗

### 各向异性过滤

- 该新选项可改善远处或斜角观看方块时的画面质量
- 默认值取决于你的图形预设，可选值为“关闭”、“2x”、“4x”和“8x”
- 若你的硬件不支持，则不会启用
- 会显著提升显存占用（尤其与较高的 mipmap 级数同时使用时）

### 环境属性

#### ARGB 颜色值的修饰符

- `override`
  - 参数格式：ARGB 颜色
- `add` —— 分量逐项相加的颜色混合
  - 参数格式：RGB 颜色
- `subtract` —— 分量逐项相减的颜色混合
  - 参数格式：RGB 颜色
- `multiply` —— 分量逐项相乘的颜色混合
  - 参数格式：RGB 或 ARGB 颜色
  - 若提供 ARGB，则 α 分量会与其他颜色通道一样独立相乘
- `alpha_blend` —— 常规 α 混合（图像编辑软件常见）
  - 参数格式：ARGB 颜色
  - 当参数 α 为 `1` 时，行为等同 override（不进行混合）

### 槽位源

- 新增“槽位源”，允许在数据包中指定任意物品栏槽位的位置
- 格式：对象，字段如下
  - `type`: 槽位来源类型
  - `<type-specific>`: 取决于类型的附加字段

#### `minecraft:empty` 类型

- 空选择，不包含任何槽位

#### `minecraft:group` 类型

- 将多个槽位来源合并为一个，结果选择包含每个提供的槽位源中的全部槽位
  - 若某槽位被多个源包含，它会在结果中重复
  - 例如 `[a, b] + [c, a]` -> `[a, b, c, a]`
- 格式：
  - `terms`: 要合并的槽位源列表
- 也可直接以内联的槽位源列表形式书写

#### `minecraft:slot_range` 类型

- 从实体或方块实体的物品栏中，选择一个槽位范围内的槽位
- 行为与 `/item` 命令的 `from` 参数一致
- 格式：
  - `source`: 槽位源的实体或方块实体（来自掉落上下文）
    - 可取 `block_entity`、`this`、`attacking_entity`、`last_damage_player`、`direct_attacker`、`target_entity`、`interacting_entity`
  - `slots`: 槽位范围，格式 `<slot_type>` 或 `<slot_type>.<slot_number>`（如 `armor.chest` 或 `container.*`）

#### `minecraft:contents` 类型

- 从一个或多个物品的“物品栏组件”中，选择所有非空槽位
  - 若该组件中未存放物品，结果选择为空
- 通过另一个槽位源指定要取其物品栏组件的物品位置
  - 若该槽位源包含多个带该组件的物品，结果将与 `minecraft:group` 类型相同方式合并
  - 例如 `Bundle [a, b] + Shulker Box [c, d]` -> `[a, b, c, d]`
- 格式：
  - `component`: 目标物品栏组件
    - 允许取值：`minecraft:bundle_contents`、`minecraft:charged_projectiles`、`minecraft:container`
  - `slot_source`: 包含目标物品所在槽位的槽位来源

#### `minecraft:filtered` 类型

- 对所选槽位应用过滤器，剔除不匹配的槽位
- 格式：
  - `item_filter`: 针对每个槽位中物品匹配的物品谓词
  - `slot_source`: 要被过滤的槽位来源

#### `minecraft:limit_slots` 类型

- 限制提供的槽位数量，结果选择最多包含该数量的槽位
  - 使数量超过上限的槽位将按包含顺序被剔除
  - 例如 `[a, b, c, d]` -> 若上限为 `3`，结果为 `[a, b, c]`
- 格式：
  - `limit`: 整形，结果中最多包含的槽位数
  - `slot_source`: 要限制的槽位源

示例：从实体的快捷栏与护甲槽位中，选择数量大于 16 的槽位：

```JSON
{
    "type": "minecraft:filtered",
    "item_filter": {
        "count": {
            "min": 16
        }
    },
    "slot_source": [
        {
            "type": "minecraft:slot_range",
            "source": "this",
            "slots": "hotbar.*"
        },
        {
            "type": "minecraft:slot_range",
            "source": "this",
            "slots": "armor.*"
        }
    ]
}
```
