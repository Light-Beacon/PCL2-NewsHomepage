### 时间线

时间线是一个新的实验性概念，定义在数据包的 `timeline` 注册表下（即位于 `data/<namespace>/timeline/<id>.json`）。

时间线基于绝对昼夜时间（即 `/time set` 所设置的时间）来指定游戏行为。时间线可选地指定一个周期，使之循环重复。

时间线的主要用途是作为环境属性的一个来源，从而对环境属性应用修饰符，以修改游戏行为或视觉效果。

格式：包含以下字段的对象

- `period_ticks`：可选整数，定义时间线的循环周期刻数
  - 若未指定，则该时间线不循环
- `tracks`：可选映射，将环境属性 ID 映射到对应的环境属性轨迹对象
  - 详见下文环境属性相关章节

时间线默认不会生效，除非被某个维度引用 —— 见下文“维度类型”。

引入时间线后，环境属性来源的处理顺序如下（从低到高优先级）：

- 维度
- 生物群系
- 时间线
- 天气 (未数据驱动)

#### 插值

时间线可以定义属性轨迹 —— 通过关键帧描述某个环境属性的修饰符随时间如何变化。依据属性不同，关键帧之间可进行插值；若不使用插值，则始终选取前一个关键帧的值。

例如，一个周期为 `24000` 的时间线可以这样设置：

- 当时间 = `0`, 时间线设置 `sky_color = #ff0000` (红色)
- 当时间 = `1000`, 时间线设置 `sky_color = #ff0000` (红色)
- 当时间 = `6000`, 时间线设置 `sky_color = #ff00ff` (品红色)

在 `0` 到 `1000` 之间，天空颜色保持红色；在 `1000` 到 `6000` 之间，将从红色平滑过渡到品红色；随后从 `6000` 开始直到时间线循环回到 `0`，颜色会缓慢回到红色。

注意：不同于生物群系，若时间线使用的是修饰符而非覆盖，插值会作用于修饰符的参数，而不是最终修改后的数值。

#### 属性轨迹

时间线中的属性轨迹为如下字段的对象：

- `ease` —— 可选缓动类型，用于在关键帧之间对数值插值进行缓动（见下文）
  - 默认：`linear`
  - 若目标属性不支持插值，则缓动类型不生效
- `keyframes` —— 关键帧对象列表，必须按 `ticks` 升序：
  - `ticks` —— 整数，位于 `0` 到 `period_ticks`（若已指定）之间，表示该关键帧生效的刻
  - `value` —— 修饰符参数（格式取决于所选修饰符）
    - 若未指定 `modifier`（或为 `override`），该字段类型与环境属性本身一致
    - 此值如何使用取决于修饰符类型
  - 注意：同一刻最多可放置两个关键帧，以实现瞬时切换
- `modifier` —— 可选 字符串修饰符 ID，依属性类型而定（见下文“通用修饰符”）
  - 默认：`override`

例如，下面这个时间线的周期为 `24000`，仅包含一个对 `minecraft:gameplay/cat_waking_up_gift_chance` 进行修改的属性轨迹。由于缓动类型为 `constant`，因此在 `362` 到 `23667` 之间取值为 `0.0`，而在 `23667` 到下一个 `362` 之间取值为 `0.7`。

与其他环境属性来源一样，轨迹中指定的数值会作为 `modifier` 的参数，叠加在较低优先级来源之上。本例中修饰符为 `maximum`，因此该时间线只会把 `cat_waking_up_gift_chance` 的取值提高到轨迹所给的数值（不会降低）。

```JSON
{
    "period_ticks": 24000,
    "tracks": {
        "minecraft:gameplay/cat_waking_up_gift_chance": {
            "ease": "constant",
            "modifier": "maximum",
            "keyframes": [
                { "ticks": 362,   "value": 0.0 },
                { "ticks": 23667, "value": 0.7 }
            ]
        }
    }
}
```

缓动类型用于通过一条曲线控制两个关键帧之间的插值。内置的缓动类型有：

- `constant` - 永远选择前一个关键帧
- `linear`
- `in_back`
- `in_bounce`
- `in_circ`
- `in_cubic`
- `in_elastic`
- `in_expo`
- `in_quad`
- `in_quart`
- `in_quint`
- `in_sine`
- `in_out_back`
- `in_out_bounce`
- `in_out_circ`
- `in_out_cubic`
- `in_out_elastic`
- `in_out_expo`
- `in_out_quad`
- `in_out_quart`
- `in_out_quint`
- `in_out_sine`
- `out_back`
- `out_bounce`
- `out_circ`
- `out_cubic`
- `out_elastic`
- `out_expo`
- `out_quad`
- `out_quart`
- `out_quint`
- `out_sine`

你也可以使用三次贝塞尔曲线指定缓动类型，格式：`{ "cubic_bezier": [ x1, y1, x2, y2 ] }`

- `x1`: 在 `0` 和 `1` 之间的浮点数, 第一个控制点的 x 轴坐标
- `y1`: 浮点数，第一个控制点的 y 轴坐标
- `x2`: 在 `0` 和 `1` 之间的浮点数, 第二个控制点的 x 轴坐标
- `y2`: 浮点数，第二个控制点的 y 轴坐标

> [!DEVNOTE]想要尝试并可视化各种缓动，[easings.net](https://easings.net/) 与 [cubic-bezier.com](https://cubic-bezier.com/) 是很好的参考！

#### 修饰符

- 新增 `blend_to_gray` RGB/ARGB 修饰符 —— 先将颜色转换为灰度，应用亮度修饰，再按给定因子与原色混合
  - 其中 `gray = brightness * (0.3 * red + 0.59 * green + 0.11 * blue)`，`result = lerp(factor, subject, [gray, gray, gray])`
  - 参数格式：含有下列字段的对象：
    - `brightness`：`0` 到 `1` 的浮点数，施加到灰度值的倍率
    - `factor`：`0` 到 `1` 的浮点数，用于混合的因子

    - `end`（原 `minecraft:end` 效果 ID）
    - 默认：`overworld`
  - `cardinal_light` —— 影响方块的基本光照方向，取值：
    - `default`（原 `minecraft:overworld` 与 `minecraft:end` 效果 ID）
    - `nether`（原 `minecraft:nether` 效果 ID）
    - 默认：default
- 字段 `fixed_time` 已由布尔值 `has_fixed_time` 取代（默认 `false`）
  - 基于时间的效果（如太阳角度）现由环境属性指定
  - 其它原先因 `fixed_time` 存在而改变的行为，现改为依据 `has_fixed_time` 布尔值
